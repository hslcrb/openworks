🗂️ 핵심 기능
1. 로컬 파일 직접 접근

지정한 폴더의 파일을 직접 읽기·편집·생성 — 수동 업로드/다운로드 불필요 Claude
파일 정리, 이름 변경, 폴더 구조 생성 등 파일 시스템 작업 전반

2. 멀티스텝 자율 실행

요청을 분석 → 계획 수립 → 서브태스크로 분화 → VM 환경에서 실행 → 결과물을 파일 시스템에 직접 전달 Claude
사용자가 자리를 비워도 알아서 완료

3. 서브에이전트 병렬 처리

독립적인 여러 태스크를 동시에 여러 Claude 인스턴스가 병렬 처리 Substack
예: 경쟁사 5곳 동시 조사 → 각각 분석 파일 생성

4. 프로페셔널 문서 생성 (Skills)

xlsx, pptx, docx, pdf 포맷 네이티브 지원 Substack
PDF는 단순 읽기 외에 병합·분리·폼 작성까지 가능 Substack
실제로 열고 바로 사용 가능한 수준의 완성본 생성


🔌 연결 기능
5. MCP 커넥터

Slack, Notion, Gmail, GitHub 등 외부 서비스 연결 Every
Google Drive MCP로 Google Docs 직접 읽기·편집 Substack
로컬 파일 작업과 외부 서비스를 하나의 워크플로우로 결합

6. Claude in Chrome 연동

브라우저를 열고 웹사이트 스크롤, 데이터 추출 후 로컬 파일과 결합 Every
예: "경쟁사 5곳 가격 조사 후 우리 스프레드시트와 비교 분석"


⚙️ 커스터마이징
7. 글로벌/폴더별 지시사항

매 세션에 적용되는 글로벌 지시사항 + 특정 폴더 작업 시에만 적용되는 폴더별 지시사항 설정 가능 Claude Cowork
선호하는 톤, 포맷, 역할 배경 등 영구 저장

8. 플러그인 (Skills 확장)

출시 후 1월 30일 플러그인 기능 추가 Every
커스텀 Skill 빌더로 자체 플러그인 제작 가능


🔒 안전 메커니즘

중요한 동작 전 사용자 승인 요청, 접근 허가한 폴더 외 접근 불가 Claude
코드 실행은 격리된 VM 환경에서 실행 (단, 파일 변경은 실제 반영됨) Claude
